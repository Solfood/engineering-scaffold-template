# Implementation Playbook

## Lifecycle stages

1. `DISCOVER`
- Confirm baseline behavior and constraints.
- Perform prior-art scan before net-new design.

2. `DECIDE`
- Compare options and choose candidate approach.
- Include explicit `reuse vs build vs buy` reasoning.

3. `BUILD`
- Implement smallest vertical slice first.

4. `VERIFY`
- Run relevant tests/checks.
- Attach evidence artifacts.
- Do not treat prose-only reasoning as verification.

5. `CONSOLIDATE`
- Promote stable outcomes into `DEC`/architecture/product docs.

6. `HANDOFF`
- Write session log with exact next actions.

## Decision communication standard

Include in user-facing updates:
- What decision was made
- Why this option was chosen
- Alternatives considered and rejected
- Evidence supporting the call
- What remains unverified
