# Documentation System

Lean documentation system for fast experiments and progressive refinement.

## Folder map

- `roadmap/`: prioritized work and sequencing
- `experiments/`: short-lived experiments and outcomes
- `decisions/`: durable decisions and rationale
- `architecture/`: current technical truth
- `product/`: product scope and requirements
- `engineering/`: lifecycle, markers, and handoffs
- `dev/`: developer setup
- `data/`: persistence/storage behavior

## Evidence policy

- Outcomes must be backed by verifiable evidence, not prose-only reasoning.
- Acceptable evidence: tests, logs, metrics, benchmark output, or reproducible command output.
- Each `Result` section links to concrete artifact(s).
